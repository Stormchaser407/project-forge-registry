# project-forge-registry - Roadmap

## Near Term

- Expand documentation enrichment to selected approved projects using the same showroom pattern.
- Run controlled markdown-only sync for additional approved slugs, one slug at a time.
- Continue validating classification outcomes and manual-review flags.

## Next Phase Targets

- Phase 5: define controlled doc sync between Obsidian `_export` and repository docs.
- Phase 6: define GitHub/Codeberg remote policy with explicit safety gates and approval flow.
- Phase 7: introduce project-sync automation with strict scope controls and reporting.

## Acceptance Signals

- Full test suite stays clean and repeatable.
- Sync reports clearly show planned vs applied actions.
- No sensitive or non-doc file movement into Obsidian lanes.
- Special-case protections (Cerberus and similar) remain intact.

## Long Horizon

- Extend policy controls from single-project operations to curated multi-project waves.
- Add stronger operator dashboards and after-action reporting for sync runs.
- Keep documentation discoverable enough that handoffs require minimal tribal context.
